# 案例2：端口无法访问（防火墙/进程未监听）

## 现象
- 特定端口无法访问
- 服务启动失败
- 防火墙阻止

## 排查步骤
1. 检查端口监听：`netstat -tuln | grep <端口>`
2. 检查防火墙规则：`iptables -L`
3. 检查服务状态：`systemctl status <服务>`

## 根因分析
- 进程未监听端口
- 防火墙规则阻止
- 服务未启动

## 解决方案
- 启动服务：`systemctl start <服务>`
- 开放端口：`iptables -A INPUT -p tcp --dport <端口> -j ACCEPT`

## 预防措施
- 配置服务自启动
- 优化防火墙规则
- 监控服务状态
